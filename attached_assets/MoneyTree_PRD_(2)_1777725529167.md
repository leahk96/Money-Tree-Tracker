# Money Tree — Product Requirements Document (MVP)

## 1. Product Overview

**Product name:** Money Tree (working title)

**One-liner:** A gamified budget tracker that turns saving money into watching a tree grow.

**Vision:** Most budget apps feel like spreadsheets or accounting software. Money Tree feels like a game. Users save money, hit goals, watch their tree grow, earn gold coins, and unlock a gold bullion. The fun is the point.

**Target user:** People aged 22-40 who have tried and abandoned traditional budgeting (YNAB, Mint, Excel) because it felt boring or punishing. They want something visual, encouraging, and habit-forming.

**Differentiator:** Every other budget app is about restriction. This one is about growth. The tree is a positive reinforcement loop, not a guilt trip.

---

## 2. Goals & Success Metrics

### Primary goals (MVP)
1. User can track their monthly budget (income, bills, expenses, savings) without friction
2. User feels rewarded when they hit savings goals (visual + emotional)
3. User returns to the app each month (habit formation)

### Success metrics
- **Activation:** % of users who complete a full month's data within 7 days of signup
- **Retention:** % of users who return in month 2, month 3
- **Engagement:** Average days between visits
- **Goal hit rate:** % of months where users hit their savings goal

### Non-goals (explicitly out of scope for MVP)
- Bank account integration (Plaid, Open Banking)
- Multiple trees / multiple savings goals
- Investment tracking
- Bill reminders / due date notifications
- Social features
- Mobile app (web-responsive only)
- Multi-currency (GBP only for MVP — make currency a constant variable for easy swap later)

---

## 3. User Stories

### Core flows

**As a new user, I want to:**
- Sign up with email + password in under 30 seconds
- See a clear empty state that explains what to do first
- Set my monthly savings goal
- Enter my income and expenses without confusion

**As a returning user, I want to:**
- Log in and immediately see my current month's status
- See my tree's current state at a glance
- Quickly add a transaction or update a number
- See my progress this month, this quarter, this year

**As an engaged user, I want to:**
- See my streak (consecutive months hitting goal)
- See visual progress charts
- Feel proud when I unlock a coin
- Add a "why" to my saving (goal name + optional photo) so it feels personal

## 4. Feature Specifications

### 4.1 Authentication
- **Sign up:** Email + password (Supabase Auth)
- **Log in:** Email + password
- **Password reset:** Email link
- **Session:** Persist across browser sessions ("remember me" by default)
- No social login for MVP (keep it simple)

### 4.2 Onboarding (first-time experience)
After signup, walk through 3 screens:
1. "What's your monthly savings goal?" → input field, prefill with £500
2. "What are you saving for?" → goal name + optional photo upload (e.g., "Bali 2026")
3. "Let's set up this month" → land them on current month's budget page with empty state guidance

### 4.3 Monthly budget page

**Layout:** Single-column on mobile, two-column on desktop. Clean and inviting, NOT spreadsheet-like. NO tables. Each section is its own visual card.

Each month is its own page with its own URL: `/budget/:year/:month` (e.g., `/budget/2026/3` for March 2026). Users can bookmark, refresh, or open multiple months in different tabs.

**Top of page — Month navigation:**
- Arrow navigation: `← March 2026 →` at the very top
- Click left/right to flip between months one at a time

**Top of page — Savings Goal Banner (the hero element):**

The savings goal is the most prominent thing on the page. It sits directly below the month navigation, above everything else. Users should see their goal status every time they open the page.

- Large banner with gold gradient (in progress) or green gradient (goal met)
- Big number display: "£300 / £500" (saved / goal)
- Progress bar showing % to goal
- Status message: "✓ Goal met! Your tree is growing" (met) or "£200 to go" (in progress)
- "Edit" button on the right to change the goal mid-month if needed
- Confetti animation the first time the goal is hit each month

**Summary cards (below the goal banner):**
- Two stat cards in a row:
  - **Income this month** (green): total income
  - **Total spent** (red/coral): total bills + needs + wants

**Budget sections (cards, NOT tables):**

CRITICAL: Each section must be its own visual card. NO tables with rows of "Category | Budget | Actual" columns. This must NOT look like Excel.

Each section card has:
- Colored left border matching the section theme (3px wide)
- Section title with emoji + name (e.g., "🌱 Savings", "💰 Income")
- Section subtotal in the top right
- Line items as simple rows: name on the left, amount on the right (right-aligned, editable)
- "+ Add row" button at the bottom of each section
- "..." menu on each row → Rename, Delete

**Section ORDER (savings first to reinforce savings-first mindset):**
1. **Savings** (forest green theme, subtle green tint background to emphasize) — Emergency fund, Investments, Other
2. **Income** (green theme) — Paycheck #1, Paycheck #2, Side income, Other
3. **Bills** (coral/pink theme) — Rent/Mortgage, Utilities, Phone/Internet, Insurance, Subscriptions, Other
4. **Needs** (blue theme) — Groceries, Transport, Medical, Personal care
5. **Wants** (amber theme) — Eating out, Shopping, Entertainment, Hobbies
6. **Debt payments** (gray theme) — Credit cards, Loans, Other

**Inline editing:**
- Click any number to edit
- Tab to next field
- Auto-save on blur (no save button)
- Number input with currency prefix (£)
- Debounce save by 500ms

**Custom rows:**
- "+ Add row" at bottom of each section
- Prompt for row name → adds new editable row
- Custom rows persist across all months (added to user_default_rows)
- Right-click or "..." menu to rename/delete

**Visual reference:**
- Should feel like Notion, Linear, Copilot Money, or Monarch Money — clean, card-based, friendly
- Should NOT feel like Excel, Google Sheets, or YNAB's web app — those are spreadsheet-y

### 4.4 Money Tree page

**Hero section — the tree (continuous, growing):**

The tree is ONE persistent visual that grows over the year. It is NOT 12 separate trees. It is NOT reset per month. It accumulates across the whole year.

- A small trunk in a pot is ALWAYS visible from day 1 (the starting state)
- Each month the user hits their savings goal → tree grows bigger (more foliage, more height, more leaves)
- Dollar bill ($) symbols appear on the green leaves and accumulate as the tree grows
- By month 12 (if all goals hit) → full mature tree covered in green leaves and dollar bills
- If user misses a month → tree stays the same that month (no growth, but doesn't shrink)
- Smooth growth animation (Framer Motion, 1.5s ease-out) when a new month's goal is hit
- Leaves sway gently (subtle infinite CSS animation)
- Display label below tree: "Your tree — X/12 months on track 🌱"

**Growth stages based on cumulative goals met this year:**
- 0 goals met: just the trunk and pot (starting state)
- 1-2 goals: small foliage, a few leaves, 1-2 dollar bills
- 3-4 goals: medium foliage, more leaves, 4-6 dollar bills
- 5-6 goals: bigger tree, fuller foliage, 8-10 dollar bills
- 7-8 goals: large tree, dense leaves, 12-15 dollar bills
- 9-10 goals: very large tree, lots of branches, 18-20 dollar bills
- 11-12 goals: full money tree, maximum size, covered in dollar bills

**Gold coins section (SEPARATE from the tree, displayed below or beside it):**

The coins are a separate achievement system from the tree itself. They live in their own section of the page.

- 4 coin slots displayed in a row: Q1, Q2, Q3, Q4
- A coin is earned when the user hits their savings goal in all 3 months of that quarter:
  - Q1 coin: Jan + Feb + Mar all met
  - Q2 coin: Apr + May + Jun all met
  - Q3 coin: Jul + Aug + Sep all met
  - Q4 coin: Oct + Nov + Dec all met
- Locked coins: gray, faded, with subtle outline
- Earned coins: gold gradient with subtle pulse animation, "ding" pop animation when first earned
- Tooltip on each: "Q1 (Jan-Mar): Hit your goal all 3 months"

**Gold bullion section (separate, below the coins):**
- Locked: gray bar saying "Earn all 4 coins to unlock"
- Unlocked (all 4 coins earned): gold gradient bar with "🏆 Gold bullion unlocked!" + confetti animation on first unlock

**Below the tree:**

**Current streak display:**
- "🔥 3 month streak!" (consecutive months hitting goal)
- Resets on a missed month
- Best streak ever shown smaller below

**Year progress grid:**
- 12 month boxes (Jan-Dec)
- Each box shows: month name, savings amount, goal status (✓ or amount short)
- Click a box to jump to that month's budget
- Met months are green and filled
- Current month has a gold border

**Quarterly coins display:**
- 4 large coin slots (Q1, Q2, Q3, Q4)
- Locked coins are gray and faded
- Earned coins are gold with "ding" animation when earned
- Tooltip on each: "Q1 (Jan-Mar): Hit your goal all 3 months"

**Gold bullion section:**
- Locked state: gray bar with "Earn all 4 coins to unlock"
- Unlocked state: gold bar with "🏆 Gold bullion unlocked!" + confetti animation

**Goal context card:**
- Shows the user's "why" (goal name + photo from onboarding)
- "You're saving for: Bali 2026" with photo
- Total saved this year: £X / £X target
- Editable from settings

### 4.5 Analytics & charts page

**Chart 1: Saved vs Goal (line chart)**
- X-axis: months (Jan-Dec)
- Y-axis: pounds saved
- Two lines: actual saved (green), goal (dashed gray)
- Hover for exact values

**Chart 2: Spending breakdown (donut chart)**
- Current month or full year toggle
- Slices: Bills, Needs, Wants, Savings, Debt
- Colors match the section themes
- Center shows total spending

**Chart 3: Income vs Spending (bar chart)**
- 12 months on X-axis
- Two bars per month: income (green), spending (coral)
- Helps user spot months where they overspent

**Chart 4: Category trend (line chart)**
- Pick a category (e.g., "Eating out")
- See spending in that category over 12 months
- Helps identify creep

**Year summary stats card:**
- Total income, total spent, total saved
- Average monthly savings
- Best month, worst month
- Days until end of year (motivational)

### 4.6 Streaks (NEW for Path A)

**What counts as a streak:**
- Consecutive months where savings >= goal
- Streak displayed prominently on Money Tree page
- "Best ever" streak tracked separately

**Streak logic:**
- Increments when current month's savings >= current month's goal
- Resets to 0 if any month is missed (in chronological order)
- A "current month" only counts toward streak after the month ends OR if user has already saved enough mid-month
- Show as: "🔥 3 month streak! Best: 5"

**Streak rewards (visual only for MVP):**
- 3 months: small flame badge
- 6 months: bigger flame
- 12 months: golden flame
- These are decorative, not mechanical

### 4.7 Goal photo & "why"

**Settings to set/edit:**
- Goal name (e.g., "Bali 2026", "House deposit", "Emergency fund")
- Goal photo (upload JPG/PNG, optional, max 5MB)
- Goal target amount (optional total target — useful for big goals)
- Yearly savings target (sum of monthly goals, auto-calculated, editable)

**Where it shows:**
- Money Tree page: "You're saving for: [name]" with photo
- Onboarding: collected on screen 2
- Editable from settings page anytime

### 4.8 Settings page

**Account:**
- Email (read-only)
- Change password
- Delete account (with confirmation)

**Goal:**
- Edit goal name
- Edit/replace goal photo
- Edit yearly target

**Defaults:**
- Default monthly savings goal (used for new months)
- Default categories (user can hide categories they don't use)

**Data:**
- Export to CSV (one click)

**Preferences:**
- Currency symbol (just £ for MVP, but make it a setting for future)

## 5. Data Model (Supabase / Postgres)

### Tables

**users** (managed by Supabase Auth)
- id (uuid, pk)
- email
- created_at

**profiles**
- user_id (uuid, fk → users.id, pk)
- goal_name (text)
- goal_photo_url (text, nullable)
- goal_target_total (numeric, nullable)
- yearly_target (numeric)
- default_monthly_goal (numeric, default 500)
- best_streak (int, default 0)
- created_at

**months**
- id (uuid, pk)
- user_id (uuid, fk)
- year (int)
- month (int, 1-12)
- savings_goal (numeric)
- created_at, updated_at
- UNIQUE constraint on (user_id, year, month)

**line_items**
- id (uuid, pk)
- month_id (uuid, fk → months.id)
- section (enum: 'income', 'bills', 'needs', 'wants', 'savings', 'debt')
- name (text)
- amount (numeric)
- sort_order (int)
- is_custom (bool, default false)
- created_at, updated_at

**user_default_rows** (templates for new months)
- id (uuid, pk)
- user_id (uuid, fk)
- section (enum)
- name (text)
- sort_order (int)

When a new month is created, populate line_items from user_default_rows.

### Row Level Security (RLS)
- All tables: users can only read/write their own data
- Enforce via Supabase RLS policies on user_id column

## 7. Design Principles

1. **Joyful, not clinical** — Use warm colors, smooth animations, encouraging copy
2. **Easy to scan** — Card-based layout, clear hierarchy, no overwhelming tables
3. **Mobile-first** — Most users will check on phone
4. **Encouraging copy** — "You saved £400 this month! 🌱" not "Savings: £400.00"
5. **One thing per screen** — Don't cram everything onto one page
6. **Forgive mistakes** — Easy undo, no destructive actions without confirmation

### Visual style
- **Primary palette:** Forest green (#228B22), money green (#85BB65), gold (#FFD700)
- **Accent palette:** Coral pink (#E8B4B8) for bills, soft blue (#7FB3D5) for needs, warm amber (#F5B041) for wants
- **Backgrounds:** Off-white (#FDFBF7), very light pink (#FDF6F6) for cards
- **Typography:** Sans-serif (Inter, Plus Jakarta Sans, or system font), generous line-height, no all-caps
- **Corners:** Rounded (12px for cards, 8px for buttons)
- **Animations:** Subtle, never jarring. Tree leaves sway. Coins ding when earned. Confetti on big wins.

## 8. User Flow (key paths)

### Path: New user signs up
1. Land on marketing page → "Get started" CTA
2. Sign up with email + password
3. Onboarding screen 1: Set monthly goal (default £500)
4. Onboarding screen 2: What are you saving for? (name + optional photo)
5. Onboarding screen 3: Brief tour of the app (3 tooltips: budget, tree, analytics)
6. Land on current month's budget page

### Path: Returning user logs in
1. Land on log in page
2. Enter credentials
3. Land on Money Tree page (their dashboard)
4. See current month status, tree state, streak

### Path: User adds a custom expense row
1. On budget page, scroll to relevant section (e.g., Bills)
2. Click "+ Add row"
3. Type name (e.g., "Gym membership")
4. Enter amount
5. Press Tab or click away — auto-saves
6. Row now appears in this month and all future months (added to defaults)

### Path: User hits their goal
1. User enters savings amount that meets/exceeds goal
2. Brief confetti animation on budget page
3. Goal status banner turns green: "✓ Goal met!"
4. Tree page now shows updated tree (one stage closer)
5. If this completes a quarter: gold coin animation on tree page

## 10. Open Questions / Decisions

1. **Should historical months be locked or editable?** Recommendation: editable, but with a warning.
2. **What happens to streak if a user adds data retroactively?** Recommendation: recalculate based on all data.
3. **Should the tree reset each year or carry over?** Recommendation: tree resets each January, but user keeps their "best ever" stats.
4. **Free or paid?** Recommendation: free for MVP, validate first. Add paywall (£3-5/month or £20/year) once you have ~100 active users.
